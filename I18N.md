# Internationalization (i18n)

Synapse supports **English** and **Greek** at the UI shell level. Lesson and note content follow the uploaded material language; UI chrome is **~68% translated** (June 2026, post Wave A/B).

## Configuration

**Settings → Language** → `UserSettings.language`: `'en' | 'el'`

Stored in `synapse:session-v2` with other preferences.

## Architecture

| Module | Role |
| ------ | ---- |
| `src/lib/i18n.ts` | Translation keys (~340+ keys), `t(key, lang)`, `I18nContext` |
| `src/lib/localeFormat.ts` | `el-GR` / `en-US` dates, relative time, heatmap tooltips |
| `src/lib/feynmanOutline.ts` | Default Feynman outline + placeholder via `t()` |
| `App.tsx` | Provides `I18nContext` to all views |
| `Shell.tsx` | Sidebar labels via `t('dashboard')`, etc. |

## Hardcoded English (intentional / low priority)

- `src/demo/*` — demo only, gated by `showDemoContent`
- `domainContent.ts` — legacy; avoid in production path
- LLM system prompts — English by design
- Server API error strings — localize in Wave C2+

## Adding translations

1. Add key to `EN` / `EL` in `src/lib/i18n.ts`
2. Replace `isEl ? '…' : '…'` with `t('yourKey')` or `useI18n().t('yourKey')`
3. Update this file when an area reaches ✅
